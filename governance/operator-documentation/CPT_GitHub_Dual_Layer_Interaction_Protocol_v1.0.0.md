# Crypto Portfolio Tracker (CPT)

## GitHub Dual-Layer Interaction Protocol

**Version:** v1.0.0 **Status:** Operational Governance Protocol
**Date:** 2026-03-02

------------------------------------------------------------------------

# 1️⃣ Purpose

This document defines the interaction protocol between:

-   CPT --- GitHub & Repository Operations
-   CPT --- GitHub Enforcement Mode

It establishes deterministic separation between:

-   Design & iteration
-   Compliance & enforcement

This protocol aligns with CPT Constitutional Charter v1.0.0.

------------------------------------------------------------------------

# 2️⃣ Role Separation (Non-Negotiable)

## 🧪 CPT --- GitHub & Repository Operations

Purpose: Design, evolve, and prototype GitHub workflows.

Allowed: - CI design - Workflow improvements - Branch protection
proposals - Commit discipline proposals - SSH strategy discussion -
Tagging strategy design - Multi-PC sync refinement

Not allowed: - Constitutional drafting - Schema redesign - Enforcement
decisions - Final lock-in of hard policies

Think: Design Lab

------------------------------------------------------------------------

## 🔒 CPT --- GitHub Enforcement Mode

Purpose: Harden, validate, and enforce Git discipline.

Allowed: - Approve branch protection configuration - Enforce commit
structure - Reject governance mutations - Approve workflow locking -
Validate semantic version tagging - Audit remote integrity - Harden CI
strictness

Not allowed: - Brainstorming - Open-ended redesign - Experimental
workflow changes

Think: Compliance Gate

------------------------------------------------------------------------

# 3️⃣ Interaction Flow

## Phase A --- Proposal (Operations Chat)

All new GitHub changes begin here.

Operations must produce:

PROPOSED CHANGE\
Scope:\
Impact:\
Files Modified:\
Risk Level: LOW \| MEDIUM \| HIGH

No enforcement occurs in this phase.

------------------------------------------------------------------------

## Phase B --- Hardening Request (Manual Transfer)

Copy the finalized proposal into:

CPT --- GitHub Enforcement Mode

Prefix with:

ENFORCEMENT REVIEW REQUEST

Proposal Origin: CPT --- GitHub & Repository Operations\
Proposed Change Summary:\
Risk Level:\
Expected Enforcement Outcome:

------------------------------------------------------------------------

## Phase C --- Enforcement Decision

Enforcement Mode must respond with one of:

APPROVED --- Ready for Implementation

or

REJECTED --- Non-Compliant with Enforcement Rules

or

CONDITIONAL APPROVAL --- Requires Modifications: - Item 1 - Item 2

No redesign.\
No new ideas.\
Only validation.

------------------------------------------------------------------------

# 4️⃣ Commit Discipline Rule

If Enforcement Mode approves:

-   Implement in repo
-   Commit must reflect enforcement approval

Example:

ci(enforcement): enable commit linting (approved by GitHub Enforcement
Mode)

------------------------------------------------------------------------

# 5️⃣ Escalation Model

If Operations and Enforcement conflict:

Enforcement wins.

Reason: Enforcement Mode protects: - Constitutional baseline - Schema
integrity - Deterministic guarantees - Release discipline

Operations is advisory. Enforcement is authoritative.

------------------------------------------------------------------------

# 6️⃣ Maturity Trigger (Future)

When CPT reaches:

-   Stable CLI surface
-   Stable snapshot model
-   First semantic version release (v1.0.0)
-   Financial reporting reliability

Then:

Enforcement Mode becomes primary.\
Operations becomes experimental workspace.

------------------------------------------------------------------------

# 7️⃣ Hard Prohibitions

Operations may NOT: - Recommend force-pushes to main - Suggest bypassing
CI - Suggest modifying locked constitutional files

Enforcement may NOT: - Draft new workflows from scratch - Propose new
architectural concepts - Introduce new Git patterns

------------------------------------------------------------------------

Idea → Operations → Formal Proposal\
↓\
Manual Transfer\
↓\
Enforcement Review\
↓\
Approved? → Implement + Commit\
Rejected? → Refine in Operations
